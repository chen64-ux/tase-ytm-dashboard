# הורדה אוטומטית של נתוני ניירות ערך מהבורסה

## שלב 1: בדיקה ידנית ראשונית

פתח/י שורת פקודה (`cmd` או `PowerShell`) בתיקייה הזו, והרץ/י:

```
pip install requests
python download_tase_data_requests.py
```

**אם זה עובד** (נוצר קובץ CSV בתיקיית `downloads/`) — מעולה, זו הגרסה הקלה
והמהירה, ואפשר לדלג ישר לשלב 3 (תזמון).

**אם זה נכשל** (הודעת שגיאה, סטטוס לא 200, או תוכן שלא נראה כמו CSV) —
עוברים לגרסת Playwright:

```
pip install playwright
playwright install chromium
python download_tase_data_playwright.py
```

זו גרסה שמריצה דפדפן Chrome אמיתי "מאחורי הקלעים" (headless), ולכן עוקפת
את הגנת ה-Imperva שזיהינו בכותרות התגובה. היא איטית יותר (~10-15 שניות
במקום שנייה) אבל אמינה יותר.

## שלב 2: בדיקת תוכן הקובץ

פתח/י את קובץ ה-CSV שנוצר ב-Excel ווודא/י שהעמודות והנתונים נראים הגיוניים
(שמות ני"ע, מחירים וכו'). **שלח/י לי דוגמה של 2-3 שורות ראשונות** (אפשר
סתם להעתיק-להדביק כטקסט, גם אם בעברית הפוכה מרוב עריכת הטקסט) — כדי שאוכל
לבנות עליו את שלב הפיצול/עדכון האוטומטי לקבצי הסקילים שלך.

## שלב 3: תזמון אוטומטי (Windows Task Scheduler)

להרצה אוטומטית פעמיים ביום (10:30 ו-14:30):

1. פתח/י את **Task Scheduler** (חפש/י "תזמון משימות" / "Task Scheduler" בתפריט התחלה).
2. **Create Task** (לא "Basic Task" - כדי לאפשר שני triggers).
3. בטאב **General**: שם = "הורדת נתוני בורסה", וסמן/י "Run whether user is
   logged on or not" אם רוצים שזה ירוץ גם כשהמחשב נעול (ידרוש הזנת סיסמה).
4. בטאב **Triggers**: **New** → Daily, שעה 10:30 → OK.
   ואז שוב **New** → Daily, שעה 14:30 → OK. (סה"כ שני triggers).
5. בטאב **Actions**: **New** → Action = "Start a program":
   - Program/script: נתיב מלא ל-python, למשל `C:\Python312\python.exe`
     (לבדוק עם `where python` בשורת הפקודה)
   - Add arguments: `download_tase_data_requests.py` (או הגרסה שעבדה לך)
   - Start in: הנתיב המלא לתיקייה הזו, למשל
     `C:\Users\<שם המשתמש>\tase_downloader`
6. שמור/י (ייתכן שיבקש סיסמת Windows אם סימנת "run whether logged on or not").

**בדיקה:** לחץ/י ימני על המשימה ברשימה → **Run** — ווודא/י שנוצר קובץ חדש
בתיקיית `downloads/`.

## הערה חשובה - מה עוד חסר

הקובץ הזה יכיל נתוני מסחר גולמיים (שמות, מחירים, הצמדה, ריבית וכו').
בדקתי את הסקילים שלך:

- **`ytm-dashboard`** צריך גם עמודות מחושבות (תשואה לפדיון, Duration,
  פרמייה) שלא קיימות בהורדה הגולמית הזו — אלה מגיעות ממודל תזרים מזומנים
  נפרד שיש לך. ברגע שנראה את מבנה הקובץ (שלב 2), אפשר לבנות סקריפט שמעדכן
  **רק** את עמודת המחיר בקובץ ה-`ytm_computed.xlsx` הקיים שלך, לפי מספר
  ני"ע - זה כן ישים.
- **`corporate-bonds-cpi`** לא קשור לנתוני מסחר בכלל (הוא מושך מדד מחירים
  לצרכן מהלמ"ס) - הקובץ הזה לא רלוונטי אליו.
